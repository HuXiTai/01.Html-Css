关于学习

程序员最重要的两种能力：学习技术的能力和解决问题的能力。

如何学习：

主动参与学习可以有更好的效果，当你想学好一个技术时你要先肯花时间去琢磨它。

  • 上课的跟着思路走，多思考，有疑问要去寻找答案。

  • 上课可以进行互动，参与到课堂中来，避免打瞌睡。

  • 课堂上或课后安排的练习通常和当天的讲解内容及之前的知识有关，尝试着去想、去做。

  • 重点内容多花时间掌握，非重点内容了解即可，不需要花太多时间，更不需要钻牛角尖。

如何练习：

学习不是靠死记硬背，只有通过不断练习新技能的同时展开思考，并且演练，成果才会显现。能否做出来是检验有没有学会的唯一标准。

  • 大量练习
    • 通过做练习题可以巩固当天所学知识，了解自己学的怎么样，然后将精力放在薄弱的知识点上。
    • 第一遍做出来很困难的或者在其他人的协助下完成的，完全有必要自己再做几遍。

  • 遇到问题

    • 先尝试自己解决问题，分析原因，有个思考的过程，这样效果会更好。

    • 自己解决不了的时候，借助网络找资料，最后才是问同学/老师。

    • 找到原因再总结问题，消化知识点，融会贯通形成自己的东西。

学习心态：

许多人相信，他们的智力水平是生来注定的，学业无成是因为先天不足。但实际情况是，当学习新东西时，大脑的思维方式会发生改变，也就是说影响智力水平的因素在很大程度上是你本人掌握的。

  • 学的还行，却感觉自己学的很差，不自信。
  • 学的很一般，却自我感觉良好，迷之自信。
  • 害怕遇到问题，遇到点问题就问同学 / 老师，不愿自己去解决问题。
  • 喜欢看、记、背代码，不愿写代码 / 写了一次就觉得自己会了。
  • 万丈高楼平地起，每天都要有进步，和自己比，不要和别人比。
